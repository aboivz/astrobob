Frequently Asked Questions

Common questions about AstroBob and their answers.

General Questions

What is AstroBob?

AstroBob is a memory-powered agent toolkit that gives IBM Bob (and other AI agents) persistent semantic, episodic, and procedural memory via AstraDB. It allows agents to remember facts, past experiences, and learned procedures across sessions.

Why do I need AstroBob?

AI agents like Bob typically forget everything between sessions. AstroBob solves this by providing:

  • Persistent memory: Remembers across sessions
  • Smart retrieval: Hybrid search with reranking
  • Self-improvement: Learns from experience and writes its own skills
  • Provenance tracking: Full audit trail of what was learned and when

Is AstroBob free?

Yes! AstroBob is open source (MIT license). AstraDB offers a generous free tier that's sufficient for most individual use cases. You only pay if you exceed the free tier limits.

What's the difference between AstroBob and RAG?

RAG (Retrieval-Augmented Generation) typically retrieves from static documents. AstroBob provides dynamic, evolving memory that agents can write to, update, and learn from. It's designed for agent memory, not document search.

Can I use AstroBob without IBM Bob?

Yes! AstroBob provides:

  • CLI: Use directly from command line
  • MCP Server: Works with any MCP-compatible client (Claude Desktop, etc.)
  • Python API: Import and use in your own code
  • WxO Integration: Use with IBM WxO multi-agent systems

Memory Types

What are the three memory types?

Type Purpose Example
Semantic Facts and knowledge "Project uses FastAPI 0.104"
Episodic Events and experiences "Fixed auth bug on 2026-05-15"
Procedural How-to procedures "To add API endpoint: 1) ..."

When should I use each memory type?

  • Semantic: For stable facts that don't change often (architecture decisions, dependencies, conventions)
  • Episodic: For things that happened (bug fixes, features added, decisions made)
  • Procedural: For reusable step-by-step procedures (how to add a feature, debug a problem)

Can I convert episodic memories to procedural?

Yes! That's what the reflect() tool is for. After solving similar problems 2-3 times, use reflect() to distill the pattern into a reusable procedure.

What's the difference between semantic and procedural?

  • Semantic: "What is" (declarative knowledge) - "The API uses JWT tokens"
  • Procedural: "How to" (actionable steps) - "To authenticate: 1) Get token 2) Add to header 3) ..."

Usage Questions

How do I know what to remember?

Remember things that:

  • ✅ You'll need to reference later
  • ✅ Are project-specific conventions
  • ✅ Took time to figure out
  • ✅ You might forget
  • ❌ Are obvious or well-documented
  • ❌ Change frequently
  • ❌ Are temporary state

How many memories should I store?

There's no hard limit, but quality > quantity. Focus on:

  • 10-20 semantic memories per project (key facts)
  • 50-100 episodic memories (significant events)
  • 5-15 procedural memories (core procedures)

Should Bob remember everything it does?

No! Be selective. Remember:

  • ✅ Non-trivial tasks (importance 3+)
  • ✅ Discoveries and learnings
  • ✅ Patterns that might repeat
  • ❌ Simple file edits
  • ❌ Routine operations
  • ❌ Obvious steps

How do I organize memories across projects?

AstroBob automatically scopes memories by project (current directory name). You can also:

  • Use --project flag to specify explicitly
  • Use tags for cross-project concepts
  • Set ASTROBOB_PROJECT environment variable
  • Use different collection prefixes for complete isolation

Can I share memories with my team?

Yes! Options:

  1. Shared AstraDB: Team uses same database (share credentials securely)
  2. Export/Import: Export memories as JSON, commit to git
  3. Skills: Export procedural memories as SKILL.md files, commit to .bob/skills/

Technical Questions

How does hybrid search work?

AstroBob uses AstraDB's hybrid search which combines:

  1. Vector search: Semantic similarity using embeddings
  2. Keyword search: Traditional text matching
  3. Reranking: AI-powered result reordering

Results are then scored using importance, recency, and success rate.

What embedding model is used?

NVIDIA NV-Embed-QA-E5-V5, hosted by AstraDB. No API keys needed - it's built into AstraDB's vectorize service.

How is data stored?

Three collections in AstraDB:

  • astrobob_semantic_memory
  • astrobob_episodic_memory
  • astrobob_procedural_memory

Each uses the same schema with vector indexing, lexical search, and reranking enabled.

Is my data secure?

Yes:

  • Data stored in your AstraDB account (you control access)
  • Encrypted in transit (HTTPS) and at rest
  • Tokens never logged or exposed
  • No data sent to third parties

What's the performance like?

Typical latencies:

  • Write (remember): 50-150ms
  • Read (recall): 100-300ms
  • Hybrid search: 200-500ms

Performance depends on AstraDB region and network latency.

Can I use a different vector database?

Currently AstroBob is designed for AstraDB. Support for other databases (Pinecone, Weaviate, etc.) could be added in future versions. Contributions welcome!

What's the difference between STDIO and HTTP mode?

  • STDIO: For Bob integration (default). Bob communicates via stdin/stdout.
  • HTTP: For debugging with MCP Inspector or other HTTP clients. Runs on port 8765.

Bob Integration

Does Bob automatically use AstroBob?

No. Bob needs to be explicitly told to use MCP tools. Best practices:

  • Use the "AstroBob Builder" custom mode (created by astrobob init)
  • Explicitly ask Bob to "check memory first"
  • Use Bob skills that reference AstroBob tools

Can Bob decide when to remember things?

Yes! Bob can autonomously call remember() when it learns something important. The "AstroBob Builder" mode encourages this behavior.

How do I teach Bob to use AstroBob effectively?

  1. Use the provided Bob skills (in .bob/skills/)
  2. Give explicit instructions: "Check memory before starting"
  3. Provide examples in custom modes
  4. Let Bob learn by doing - it will improve over time

Can multiple Bob instances share memory?

Yes! If they use the same AstraDB database and project name, they'll share memories. Useful for team collaboration.

Skills Export

What are "learned skills"?

Procedural memories exported as native Bob SKILL.md files. Bob can use these directly without calling MCP tools.

When should I export skills?

Export when:

  • You have stable, high-value procedures (importance 4-5)
  • You want Bob to use procedures without MCP overhead
  • You want to commit procedures to git for team sharing

What's the difference between MCP tools and skills?

Aspect MCP Tools Skills
Storage AstraDB (dynamic) Files (static)
Updates Real-time Manual re-export
Search Hybrid search Bob's skill matching
Use Case Evolving knowledge Stable procedures

Can I edit exported skills?

Yes! They're just markdown files in .bob/skills/learned/. Edit as needed. Note: Changes won't sync back to AstraDB.

Costs & Limits

How much does AstraDB cost?

Free tier includes:

  • 25 GB storage
  • 5 million reads/month
  • 1 million writes/month
  • Unlimited vector searches

This is typically enough for individual use. See AstraDB pricing for paid tiers.

How many memories fit in the free tier?

Rough estimates:

  • ~100,000 memories in 25 GB (assuming ~250 KB per memory with vectors)
  • ~1,000 writes/day within free tier
  • ~150,000 reads/day within free tier

What happens if I exceed free tier?

AstraDB will prompt you to upgrade. You can also:

  • Clean up old memories with forget()
  • Archive to JSON and delete
  • Use separate databases per project

Advanced Usage

Can I use AstroBob programmatically?

Yes! Import the Python API:

from astrobob.core.store import MemoryStore
from astrobob.core.retriever import Retriever
from astrobob.astra.client import AstraClient
from astrobob.config import Config

config = Config()
client = AstraClient(config)
store = MemoryStore(client.get_database(), config.collection_prefix)

# Store memory
memory = MemoryDocument(...)
store.insert(memory)

# Retrieve
retriever = Retriever(store)
results = retriever.search("query")

Can I customize the ranking algorithm?

Yes! Edit src/astrobob/core/ranking.py and adjust the weights in the calculate_score() function.

Can I add custom memory types?

Not currently. The three types (semantic, episodic, procedural) are core to the design. However, you can use tags to create sub-categories.

Can I use AstroBob with WxO?

Yes! See examples/wxo/ for templates. AstroBob can be used as a tool in WxO multi-agent systems.
