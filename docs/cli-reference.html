CLI Reference

Complete reference for all AstroBob command-line interface commands and options.

💡 Tip: Use astrobob --help or astrobob --help for inline help.

Global Options

These options work with all commands:

Option Description
--version Show version and exit
--help Show help message and exit
--verbose, -v Enable verbose output
--quiet, -q Suppress non-error output

Command Overview

🚀 init

Initialize AstroBob in a project

🗄️ astra

Manage AstraDB collections

🧠 memory

Store and retrieve memories

🔌 mcp

Start MCP server for Bob

📚 skills

Export memories as Bob skills

🏥 doctor

Diagnose configuration issues

astrobob init

Initialize AstroBob in the current project directory.

Usage

astrobob init [OPTIONS]

Description

Creates the necessary directory structure and configuration files for AstroBob:

  • .bob/ - Bob configuration directory
  • .bob/mcp.json - MCP server configuration
  • .bob/custom_modes.yaml - Custom Bob modes
  • .bob/skills/ - Bob skills directory
  • .env.example - Example environment variables

Options

Option Type Default Description
--force flag false Overwrite existing files
--no-skills flag false Skip creating default skills

Examples

# Initialize in current directory
astrobob init

# Force overwrite existing files
astrobob init --force

# Initialize without default skills
astrobob init --no-skills

Exit Codes

  • 0 - Success
  • 1 - Files already exist (use --force)
  • 2 - Permission error

astrobob astra

Manage AstraDB collections and connections.

astrobob astra setup

Create required AstraDB collections.

Usage

astrobob astra setup [OPTIONS]

Description

Creates three collections in AstraDB:

  • astrobob_semantic_memory - For facts and knowledge
  • astrobob_episodic_memory - For events and experiences
  • astrobob_procedural_memory - For procedures and how-tos

Options

Option Type Default Description
--prefix string astrobob Collection name prefix
--drop-existing flag false Drop existing collections first

Examples

# Create collections with default prefix
astrobob astra setup

# Use custom prefix
astrobob astra setup --prefix myproject

# Recreate collections (WARNING: deletes data)
astrobob astra setup --drop-existing

Usage

astrobob astra status

astrobob memory

Store, retrieve, and manage memories.

astrobob memory remember

Store a new memory.

Usage

astrobob memory remember [OPTIONS]

Required Options

Option Type Description
--type choice Memory type: semantic, episodic, or procedural
--content string Memory content (what to remember)

Optional Options

Option Type Default Description
--project string current dir Project name
--importance int (1-5) 3 Importance level
--tags string none Comma-separated tags
--confidence float (0-1) 1.0 Confidence level
--source string cli Source: cli, bob, bobshell, wxo, user

Examples

# Store a semantic memory (fact)
astrobob memory remember \
  --type semantic \
  --content "Project uses FastAPI 0.104 with async routes" \
  --importance 4 \
  --tags fastapi,architecture

# Store an episodic memory (event)
astrobob memory remember \
  --type episodic \
  --content "Fixed auth bug by adding token refresh logic" \
  --importance 3 \
  --tags bugfix,auth

# Store a procedural memory (how-to)
astrobob memory remember \
  --type procedural \
  --content "To add API endpoint: 1) Define route 2) Add schema 3) Write tests" \
  --importance 5 \
  --tags api,procedure

astrobob memory recall

Search and retrieve memories.

Usage

astrobob memory recall QUERY [OPTIONS]

Arguments

Argument Type Description
QUERY string Search query (natural language)

Options

Option Type Default Description
--type choice all Filter by memory type
--project string current Filter by project
--tags string none Filter by tags (comma-separated)
--limit int 10 Maximum results to return
--min-importance int (1-5) 1 Minimum importance level
--format choice table Output format: table, json, yaml

Examples

# Search all memories
astrobob memory recall "how to add API endpoint"

# Search only procedural memories
astrobob memory recall "authentication" --type procedural

# Search with tag filter
astrobob memory recall "bug" --tags auth,security

# Get top 5 most important
astrobob memory recall "architecture" --limit 5 --min-importance 4

# Output as JSON
astrobob memory recall "fastapi" --format json

astrobob memory reflect

Create a procedural memory from episodic memories.

Usage

astrobob memory reflect [OPTIONS]

Required Options

Option Type Description
--episode-id string Episode ID (can be repeated)
--lesson string The lesson learned

Optional Options

Option Type Default Description
--project string current Project name
--importance int (1-5) 4 Importance level
--tags string none Comma-separated tags

Examples

# Create reflection from episodes
astrobob memory reflect \
  --episode-id 01HXXX... \
  --episode-id 01HYYY... \
  --lesson "Always add integration tests when fixing auth bugs" \
  --importance 4 \
  --tags testing,auth

astrobob mcp

Start the MCP server for Bob integration.

Usage

astrobob mcp [OPTIONS]

Description

Starts the Model Context Protocol (MCP) server that exposes 5 tools to Bob:

  • remember - Store new memories
  • recall - Search memories
  • reflect - Create procedural memories
  • forget - Delete memories
  • audit_trail - View memory history

Options

Option Type Default Description
--http flag false Use HTTP/SSE instead of STDIO
--port int 8765 HTTP port (only with --http)
--host string localhost HTTP host (only with --http)

Examples

# Start in STDIO mode (default, for Bob)
astrobob mcp

# Start in HTTP mode
astrobob mcp --http

# Start on custom port
astrobob mcp --http --port 9000
💡 Note: STDIO mode is recommended for Bob integration. HTTP mode is useful for debugging with MCP Inspector.

astrobob skills

Export procedural memories as Bob skills.

astrobob skills sync

Export high-value procedural memories to .bob/skills/learned/.

Usage

astrobob skills sync [OPTIONS]

Options

Option Type Default Description
--min-importance int (1-5) 4 Minimum importance to export
--project string current Filter by project
--dry-run flag false Show what would be exported
--force flag false Re-export already exported memories

Examples

# Export important procedures (importance >= 4)
astrobob skills sync

# Export all procedures (importance >= 3)
astrobob skills sync --min-importance 3

# Preview what would be exported
astrobob skills sync --dry-run

# Re-export everything
astrobob skills sync --force

astrobob doctor

Description

Runs comprehensive health checks:

  • ✓ Environment variables configured
  • ✓ AstraDB connectivity
  • ✓ Collections exist and accessible
  • ✓ MCP server importable
  • ✓ .bob/ directory structure valid

astrobob doctor

Environment Variables

AstroBob uses these environment variables (typically in .env):

Variable Required Description
ASTRA_DB_API_ENDPOINT Yes AstraDB API endpoint URL
ASTRA_DB_APPLICATION_TOKEN Yes AstraDB application token
ASTRA_DB_KEYSPACE No Keyspace name (default: default_keyspace)
ASTROBOB_COLLECTION_PREFIX No Collection name prefix (default: astrobob)
ASTROBOB_PROJECT No Default project name (default: current directory)

Tips & Best Practices

💡 Pro Tips

  • Use aliases: Add alias ab='astrobob' to your shell config
  • Tab completion: Most shells support tab completion for commands
  • JSON output: Use --format json for scripting and automation
  • Importance levels: 5=critical, 4=important, 3=useful, 2=nice-to-know, 1=trivial
  • Tag consistently: Use lowercase, hyphenated tags like api-design
