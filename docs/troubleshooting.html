Troubleshooting Guide

Solutions to common issues and problems when using AstroBob.

💡 First Step: Run astrobob doctor to diagnose most common issues automatically.

Quick Diagnostics

# Run comprehensive health check
astrobob doctor

# Check AstraDB connection
astrobob astra status

# Verify environment variables
cat .env

# Check Python version
python --version  # Should be 3.12+

Installation Issues

❌ "astrobob: command not found"

Cause

AstroBob not installed or not in PATH.

Solutions

# Verify installation
uv tool list

# If not installed
uv tool install astrobob

# Add to PATH (if needed)
export PATH="$HOME/.local/bin:$PATH"  # Linux/macOS
# Or add to ~/.bashrc or ~/.zshrc
# Verify installation
pip show astrobob

# If not installed
pip install git+https://github.com/aboivz/astrobob.git

# Check if in PATH
which astrobob

# If not found, use full path
python -m astrobob.cli.app --help
# Verify installation
pipx list

# If not installed
pipx install git+https://github.com/aboivz/astrobob.git

# Ensure pipx bin directory in PATH
pipx ensurepath

❌ "Python 3.12+ required"

Cause

AstroBob requires Python 3.12 or higher.

Solution

# Check current version
python --version

# Install Python 3.12+ (Ubuntu/Debian)
sudo apt update
sudo apt install python3.12

# Install Python 3.12+ (macOS with Homebrew)
brew install python@3.12

# Install Python 3.12+ (Windows)
# Download from python.org

# Use pyenv for version management
pyenv install 3.12.0
pyenv local 3.12.0

❌ Dependency conflicts

Cause

Conflicting package versions in your environment.

Solution

# Use uv (recommended - handles conflicts automatically)
uv tool install astrobob

# Or create isolated environment with pipx
pipx install git+https://github.com/aboivz/astrobob.git

# Or use virtual environment
python -m venv .venv
source .venv/bin/activate  # Linux/macOS
# .venv\Scripts\activate  # Windows
pip install git+https://github.com/aboivz/astrobob.git

Configuration Issues

❌ "Missing environment variable: ASTRA_DB_API_ENDPOINT"

Cause

Required environment variables not set.

Solution

# Create .env file in project directory
cat > .env << 'EOF'
ASTRA_DB_API_ENDPOINT=https://xxx-us-east-2.apps.astra.datastax.com
ASTRA_DB_APPLICATION_TOKEN=AstraCS:xxxxxxxxxxxxx
ASTRA_DB_KEYSPACE=default_keyspace
ASTROBOB_COLLECTION_PREFIX=astrobob
EOF

# Verify .env file exists
ls -la .env

# Check if variables are loaded
astrobob doctor
⚠️ Important: The .env file must be in the directory where you run astrobob commands.

❌ "No .env file found"

Cause

Running commands from wrong directory.

Solution

# Navigate to project directory
cd ~/projects/my-project

# Verify .env exists
ls -la .env

# Or specify project explicitly
export ASTROBOB_PROJECT=my-project

❌ "Authentication failed"

Cause

Invalid or expired AstraDB credentials.

Solution

  1. Go to astra.datastax.com
  2. Navigate to your database
  3. Click "Connect" → "Generate Token"
  4. Copy new token (starts with AstraCS:)
  5. Update .env file with new token
  6. Test: astrobob astra status
🔒 Security: Never commit .env to git. Add it to .gitignore.

AstraDB Issues

❌ "Connection refused" or "Timeout"

Possible Causes

  • Wrong API endpoint URL
  • Network/firewall issues
  • AstraDB database not active

Solutions

# 1. Verify endpoint URL format
# Should be: https://xxx-region.apps.astra.datastax.com
echo $ASTRA_DB_API_ENDPOINT

# 2. Test connectivity
curl -I $ASTRA_DB_API_ENDPOINT

# 3. Check database status in Astra UI
# Database should show "Active" status

# 4. Try different region (if available)
# Update ASTRA_DB_API_ENDPOINT in .env

# 5. Check firewall/proxy settings
# Ensure HTTPS (443) is allowed

❌ "Collection not found"

Cause

Collections not created yet.

Solution

# Create collections
astrobob astra setup

# Verify collections exist
astrobob astra status

# Expected output:
# ✓ astrobob_semantic_memory (0 documents)
# ✓ astrobob_episodic_memory (0 documents)
# ✓ astrobob_procedural_memory (0 documents)

❌ "Rate limit exceeded"

Cause

Too many requests to AstraDB.

Solution

# Wait a few seconds and retry

# For bulk operations, add delays
for id in $(cat memory_ids.txt); do
  astrobob memory recall "$id"
  sleep 0.5  # 500ms delay
done

# Consider upgrading AstraDB tier for higher limits

❌ "Vectorize service error"

Cause

AstraDB vectorize service issue (rare).

Solution

# 1. Check AstraDB status page
# https://status.datastax.com

# 2. Verify region supports vectorize
# Recommended: us-east-2

# 3. Recreate collections (last resort)
astrobob astra setup --drop-existing

# ⚠️ WARNING: This deletes all data!

Bob Integration Issues

❌ Bob doesn't see MCP tools

Possible Causes

  • MCP server not configured
  • Bob not in Advanced mode
  • Wrong mcp.json path

Solutions

# 1. Verify .bob/mcp.json exists
cat .bob/mcp.json

# 2. Ensure correct format
{
  "mcpServers": {
    "astrobob": {
      "command": "astrobob",
      "args": ["mcp"],
      "env": {
        "ASTRA_DB_API_ENDPOINT": "...",
        "ASTRA_DB_APPLICATION_TOKEN": "..."
      }
    }
  }
}

# 3. Test MCP server manually
astrobob mcp --http
# Then open http://localhost:8765 in browser

# 4. Restart Bob
# Close and reopen Bob application

# 5. Verify Bob is in Advanced mode
# Check Bob settings/preferences

❌ Bob tool calls fail

Cause

MCP server errors or invalid parameters.

Solution

# 1. Check MCP server logs
# Look for error messages in terminal where MCP server runs

# 2. Test tools directly
astrobob memory remember \
  --type semantic \
  --content "Test memory" \
  --importance 3

# 3. Verify environment variables in mcp.json
# Ensure they match your .env file

# 4. Run doctor
astrobob doctor

# 5. Restart MCP server
# Kill existing process and restart:
astrobob mcp

❌ Bob MCP calls are slow

Possible Causes

  • Network latency to AstraDB
  • Large result sets
  • Cold start (first query)

Solutions

# 1. Use closer AstraDB region
# Update to nearest region in Astra UI

# 2. Limit result count
recall({
  "query": "...",
  "limit": 5  # Instead of default 10
})

# 3. Filter by importance
recall({
  "query": "...",
  "min_importance": 4  # Only important memories
})

# 4. Use specific memory types
recall({
  "query": "...",
  "memory_type": "procedural"  # Instead of "all"
})

Memory Issues

❌ "No memories found" but I stored some

Possible Causes

  • Wrong project filter
  • Memories soft-deleted
  • Query doesn't match content

Solutions

# 1. Check all projects
astrobob memory report

# 2. Try broader query
astrobob memory recall "anything" --limit 50

# 3. Check specific type
astrobob memory recall "test" --type semantic
astrobob memory recall "test" --type episodic
astrobob memory recall "test" --type procedural

# 4. Verify not deleted
# Deleted memories have deleted_at timestamp

# 5. Check AstraDB Studio directly
# Log into Astra UI → Data Explorer

❌ Recall returns irrelevant results

Cause

Query too broad or memories not well-tagged.

Solutions

# 1. Use more specific queries
# Bad:  "api"
# Good: "how to add API endpoint with authentication"

# 2. Filter by tags
astrobob memory recall "api" --tags api,auth

# 3. Filter by importance
astrobob memory recall "api" --min-importance 4

# 4. Use specific memory type
astrobob memory recall "how to" --type procedural

# 5. Improve memory content
# Store more detailed, specific memories

❌ Duplicate memories

Cause

Same information stored multiple times.

Solutions

# 1. Search before storing
astrobob memory recall "topic" --type semantic

# 2. Delete duplicates
astrobob memory forget --type semantic --id 01HXXX...

# 3. Use supersedes for updates
# When storing updated version, note old ID

# 4. Regular cleanup
# Weekly review and remove outdated memories

Skills Export Issues

❌ "No skills to export"

Cause

No procedural memories meet importance threshold.

Solution

# 1. Check procedural memories
astrobob memory recall "procedure" --type procedural

# 2. Lower importance threshold
astrobob skills sync --min-importance 3

# 3. Create procedural memories
astrobob memory remember \
  --type procedural \
  --content "How to do X: step 1, step 2..." \
  --importance 4

# 4. Use dry-run to preview
astrobob skills sync --dry-run

❌ Bob doesn't use exported skills

Cause

Skills not in correct location or format.

Solution

# 1. Verify skill files exist
ls -la .bob/skills/learned/

# 2. Check SKILL.md format
cat .bob/skills/learned/*/SKILL.md

# 3. Restart Bob
# Close and reopen Bob application

# 4. Verify Bob skills directory
# Check Bob settings for skills path

# 5. Re-export with force
astrobob skills sync --force

Performance Issues

❌ Slow recall queries

Solutions

# 1. Reduce result limit
astrobob memory recall "query" --limit 5

# 2. Use specific memory type
astrobob memory recall "query" --type procedural

# 3. Filter by tags
astrobob memory recall "query" --tags specific-tag

# 4. Check AstraDB region latency
# Use region closest to you

# 5. Monitor with audit trail
astrobob memory audit --type procedural --id 01HXXX...

❌ Too many memories, slow performance

Solutions

# 1. Clean up old memories
astrobob memory recall "old" --format json | \
  jq -r '.results[] | select(.importance < 3) | .memory_id' | \
  xargs -I {} astrobob memory forget --type episodic --id {}

# 2. Archive by project
# Export memories to JSON, then delete
astrobob memory report --format json > archive.json

# 3. Use importance filters
# Only query important memories
astrobob memory recall "query" --min-importance 4

# 4. Separate projects
# Use different collection prefixes per project
ASTROBOB_COLLECTION_PREFIX=project1 astrobob astra setup

Common Error Messages

Error Meaning Fix
ConfigError Missing/invalid config Check .env, run astrobob doctor
AstraConnectionError Can't connect to AstraDB Verify credentials, check network
MemoryNotFoundError Memory ID doesn't exist Verify ID with recall()
ValidationError Invalid parameters Check parameter types/ranges
ImportError Missing dependencies Reinstall: uv tool install astrobob

Debug Checklist

Before reporting an issue, please check:

  • ✅ Ran astrobob doctor
  • ✅ Checked .env file exists and is correct
  • ✅ Verified Python version (3.12+)
  • ✅ Tested AstraDB connection: astrobob astra status
  • ✅ Checked this troubleshooting guide
  • ✅ Searched existing issues
  • ✅ Tried with --verbose flag

Reporting Bugs

When reporting issues, please include:

# 1. Version info
astrobob --version
python --version

# 2. Doctor output
astrobob doctor

# 3. Error message (with --verbose)
astrobob memory recall "test" --verbose

# 4. Environment (sanitized)
cat .env | sed 's/AstraCS:.*/AstraCS:REDACTED/'

# 5. Steps to reproduce
# Exact commands that cause the issue
