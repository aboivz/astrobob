Quick Start Guide

Get AstroBob up and running in under 5 minutes. This guide walks you through installation, configuration, and your first memory operations.

Prerequisites: Python 3.12+, AstraDB account (free tier works), and basic command line knowledge.

Step 1: Install AstroBob

Install using uv (recommended) or pip:

# Install uv if you haven't already
curl -LsSf https://astral.sh/uv/install.sh | sh

# Install AstroBob
uv tool install astrobob

# Verify installation
astrobob --version
# Install from GitHub
pip install git+https://github.com/aboivz/astrobob.git

# Verify installation
astrobob --version

Step 2: Set Up AstraDB

2.1 Create Database

  1. Go to astra.datastax.com
  2. Sign up or log in (free tier available)
  3. Click "Create Database"
  4. Configure:
    • Database name: astrobob
    • Provider: AWS
    • Region: us-east-2 (recommended)
    • Tier: Serverless (Free)
  5. Click "Create Database" and wait ~2 minutes

2.2 Get Credentials

  1. In your database dashboard, click "Connect"
  2. Copy the API Endpoint (looks like https://xxx-us-east-2.apps.astra.datastax.com)
  3. Click "Generate Token" under Application Token
  4. Select role: Database Administrator
  5. Copy the token (starts with AstraCS:...)
⚠️ Security: Save your token securely. It won't be shown again!

2.3 Configure Environment

Create a .env file in your project directory:

# Navigate to your project
cd ~/projects/my-project

# Create .env file
cat > .env << 'EOF'
ASTRA_DB_API_ENDPOINT=https://xxx-us-east-2.apps.astra.datastax.com
ASTRA_DB_APPLICATION_TOKEN=AstraCS:xxxxxxxxxxxxx
ASTRA_DB_KEYSPACE=default_keyspace
ASTROBOB_COLLECTION_PREFIX=astrobob
EOF

Step 3: Initialize AstroBob

Initialize AstroBob in your project:

# Initialize (creates .bob/ directory and config files)
astrobob init

# Set up AstraDB collections
astrobob astra setup

# Verify everything works
astrobob doctor

You should see:

✓ Environment variables configured
✓ AstraDB connection successful
✓ Collections created: semantic, episodic, procedural
✓ MCP server importable
✓ .bob/ structure valid

All systems operational! 🚀

Step 4: Store Your First Memory

4.1 Store a Semantic Memory (Fact)

astrobob memory remember \
  --type semantic \
  --content "This project uses FastAPI 0.104 with async routes" \
  --importance 4 \
  --tags fastapi,architecture

4.2 Store an Episodic Memory (Event)

astrobob memory remember \
  --type episodic \
  --content "Fixed authentication bug by adding token refresh logic in auth.py" \
  --importance 3 \
  --tags bugfix,auth

4.3 Store a Procedural Memory (How-To)

astrobob memory remember \
  --type procedural \
  --content "To add a new API endpoint: 1) Define route in routes/, 2) Add schema in schemas/, 3) Write tests in tests/api/, 4) Update OpenAPI docs" \
  --importance 5 \
  --tags api,procedure

Step 5: Recall Memories

5.1 Search All Memories

astrobob memory recall "how to add API endpoint"

5.2 Search Specific Type

# Search only procedural memories
astrobob memory recall "how to add API endpoint" --type procedural

# Search with tags
astrobob memory recall "authentication" --tags auth,security

5.3 View Memory Report

astrobob memory report

Output example:

Step 6: Integrate with IBM Bob

6.1 Start MCP Server

AstroBob includes an MCP server that Bob can connect to:

# Start in STDIO mode (default for Bob)
astrobob mcp serve

# Or start in HTTP mode (for debugging - not yet implemented)
astrobob mcp serve --transport http

6.2 Configure Bob

The astrobob init command already created .bob/mcp.json. Verify it looks like this:

{
  "mcpServers": {
    "astrobob": {
      "command": "astrobob",
      "args": ["mcp", "serve"],
      "env": {
        "ASTRA_DB_API_ENDPOINT": "https://xxx.apps.astra.datastax.com",
        "ASTRA_DB_APPLICATION_TOKEN": "AstraCS:xxxxx"
      }
    }
  }
}

6.3 Use in Bob

Open Bob in Advanced mode and try:

Bob Chat Example
You: "Check if we have any procedures for adding API endpoints"

Bob: [calls recall(query="add API endpoint", memory_type="procedural")]
     Found procedure: "To add a new API endpoint: 1) Define route..."
     
     Yes! I found our standard procedure. Would you like me to follow it?

Step 7: Create Reflections

Convert episodic memories into reusable procedures:

# First, get episode IDs
astrobob memory recall "fixed bug" --type episodic

# Create reflection from episodes
astrobob memory reflect \
  --episode-id 01HXXX... \
  --episode-id 01HYYY... \
  --lesson "Always add integration tests when fixing auth bugs" \
  --importance 4 \
  --tags testing,auth

Step 8: Export as Bob Skills

Turn high-value procedural memories into native Bob skills:

# Export all important procedures (importance >= 4)
astrobob skills sync --min-importance 4

# Check generated skills
ls .bob/skills/learned/

This creates .bob/skills/learned//SKILL.md files that Bob can use natively!

Common Workflows

Daily Development Workflow

# Morning: Check what you learned yesterday
astrobob memory recall "yesterday" --type episodic

# During work: Store important discoveries
astrobob memory remember --type semantic \
  --content "API rate limit is 100 req/min per user" \
  --importance 4

# End of day: Reflect on what you learned
astrobob memory reflect \
  --episode-id  \
  --lesson "Key insight from today's work"

# Weekly: Export new skills
astrobob skills sync

Team Onboarding Workflow

# New team member: Get project knowledge
astrobob memory recall "architecture" --type semantic
astrobob memory recall "how to" --type procedural

# Export all procedures as documentation
astrobob skills sync --min-importance 3

Debugging Workflow

# Check if similar bug was fixed before
astrobob memory recall "authentication error" --type episodic

# After fixing: Document the solution
astrobob memory remember --type episodic \
  --content "Fixed auth error by clearing Redis cache" \
  --importance 3 \
  --tags bugfix,auth,redis

# If it's a pattern: Create procedure
astrobob memory reflect \
  --episode-id  \
  --lesson "Auth errors often need cache clear" \
  --importance 4

Pro Tips

💡 Best Practices

  • Be specific: "FastAPI uses async def for routes" beats "FastAPI is async"
  • Use importance wisely: 5 = critical, 4 = important, 3 = useful, 2 = nice-to-know, 1 = trivial
  • Tag consistently: Use lowercase, hyphenated tags like api-design, bug-fix
  • Reflect regularly: Weekly reflections turn experience into reusable knowledge
  • Clean up: Use astrobob memory forget for outdated memories

Quick Troubleshooting

Issue Solution
❌ "Connection refused" Check ASTRA_DB_API_ENDPOINT in .env
❌ "Authentication failed" Verify ASTRA_DB_APPLICATION_TOKEN is correct
❌ "Collection not found" Run astrobob astra setup to create collections
❌ Bob doesn't see MCP tools Check .bob/mcp.json exists and Bob is in Advanced mode
❌ "No memories found" Verify you're in the correct project directory with .env

For more detailed troubleshooting, see the Troubleshooting Guide.
